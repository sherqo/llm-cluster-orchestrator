A.6 Start ChromaDB



The vector database is located in:

vector-db/



Start it using Docker Compose:



cd vector-db

docker compose up -d --build



This starts ChromaDB on:

http://localhost:8000



It also indexes the documents in:

vector-db/embedded-docs/



Return to the project root:

cd ..



A.7 Run the Master



Open a terminal and run:

cd master

go run main.go

The Master starts the HTTP server on:

http://localhost:8080

It also opens the terminal monitoring dashboard.



 A.8 Run the Agent



Open another terminal and run the Agent:

cd agent

go run . --master-url http://localhost:8080 --listen :9000 --advertise-host 127.0.0.1



The Agent registers with the Master, starts or checks Ollama and ChromaDB, and becomes available to create Workers.

The Agent listens to:

http://localhost:9000



A.9 Test the Chat Endpoint



After the Master and Agent are running, the Agent should create or report Workers. Then send a chat request to the Master.



Using curl:

curl -X POST http://localhost:8080/chat \

  -H "Content-Type: application/json" \

  -d '{

    "userId": "user-1",

    "prompt": "What happened on D-Day?",

    "tier": "pro"

  }'







Expected response format:

{

  "requestId": "generated-request-id",

  "reply": "Generated answer from the worker"

}



You can also test this usingTUI of the client.

C.1 Master Startup

File:

master/main.go

func main() {

    log.SetOutput(monitoring.NewStdLogWriter())

    router := lib.NewRouter()



    if router.WorkerCount() == 0 {

        fmt.Println("no workers available at startup")

    }



    go lib.Serve(router)



    if err := tui.Run(router); err != nil {

        fmt.Printf("Error starting TUI: %v\n", err)

    }

}
